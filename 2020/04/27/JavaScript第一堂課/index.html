  1. JavaScript是如何運行？
    （A）編譯式語言：

編譯式語言在預先編譯時，進行除錯。這類型語言效能也會比較好。
（B）直譯式語言（JavaScript屬於此種）：
JavaScript透過直譯器，來將程式碼轉換成電腦看得懂的語言，如此，電腦才能夠在console上面進行運行。

透過直譯器將原始碼生成電腦看得懂的語言，並在瀏覽器上運行，此時，錯誤會顯示在console上。
優點：彈性高，不須預先定義型別。

(a) 語法單元化：
將JS標點符號或詞彙，一一解析出來
(b) 抽象結構樹：
定義程式碼結構
[注意]：此部分還未運行程式碼！！
(c) 代碼生成：
代碼生成會因為執行環境不同而不同（ex：NodeJS或不同瀏覽器）
[注意]：此部分才運行程式碼

  1. 取值與賦予值的關西？
    RHS(Ride-Hand-Side)：取值來自於右側變數上
    LHS(Left-Hand-Side)：用來賦予值到左側變數上
    var man = “Edward”; －－＞代表利用RHS取值於Edward，並立用LHS賦與值到man左側
    結論：
    LHS是在編譯過程中發生的錯誤（當發現此問題時，要查看是不是有變數無法被賦與）
    RHS不會在編譯過程中出現錯誤，執行階段才會出現錯誤
    －－＞通常會出現Uncaught ReferenceError: xxx is not defined.
  2. 語法作用域（Lexical Scope）及動態作用域？
    [定義]：
    (A) 語法（靜態）作用域：變數的作用域在語法解析時，就已經確定了作用域，且不會再改變
    (B) 動態作用域：變數的作用域在函式調用時，才決定
    －－＞JavaScript是屬於語法作用域，在程式碼撰寫時，就確定作用域

此外，JavaScript的作用域是一層一層包住，如下：

上圖，兩個function的作用域是獨立的。如果今天，各自的function沒有在自己的變數上找到變數，此時，才會向外查找。如果向外查找也無，就會出現ReferenceError。

PS.(第五小節範圍鍊和此部分靜態作用域，皆為程式碼撰寫完成時，就已經確定了)
－－＞此部分，與執行環境沒關西

  1. Execution Context(執行環境)
    (A) Function宣告的時候，不會產稱執行環境，只有在呼叫的時候，才會產生執行環境。
    (B) 每個函式都有自己的執行環境，可以不斷地被呼叫，便可以不斷地產生新的執行環境，在各自的函式中，也稱為限制作用域，並且會有自己的變數。
    [注意]：函式撰寫的時候，已經確定作用域，但是要執行（呼叫），才能夠產生執行環境，那這個執行環境內，才會有屬於它自己的變數。
    (C) 當函式執行完畢後會離開執行堆疊（Execution Stack）。
    (D) 函式會產生自己的執行環境外，全域也會產生自己的執行環境。全域的執行環境是在網頁一開啟，或者 Node.js一開啟時，執行環境就已經建立了，同時，也會產稱window或global的變數，也可以用this代替。（下圖可以明確看到，this == window）

[備註]：this會隨著執行環境的不同而指向不同
(E) 執行堆疊(Execution Stack)與執行環境(Execution Context)的關西
[結論一]：執行環境可以透過呼叫不斷產生

function sayHi(name){
    var greeting = "hi";
    return greeting + "" + name;
}
function doSomething(){
    var mom = "老媽";
    console.log(1,sayHi(mom));
}
doSomething();

  1. 提升Hoisting
    執行環境分兩階段：
    （1） 創造環境（記憶體空間）－－＞提升
    (a) 變數：此時記憶體只生成此變數（準備記憶體空間），還未給予值，所以undefined
    －－＞此步驟為Hoisting
    (b) 函式表達式：此時記憶體只生成此變數（準備記憶體空間），還未給予值，所以undefined
    (c) 函式陳述式：會將整個函式，載入進來，不用等到執行（此時函式已可運行）
    （2） 執行
    (a) 變數：執行階段才賦予值
    (b) 函式表達式：執行階段才賦予值
    (c) 函式陳述式：創造階段已完成
    //通常變數是這樣:
    var Ming = “小明”;
    //其實是分兩階段:
    var Ming; //創造階段
    Ming = “小明”; //執行階段
    //所以，在此console，會出現：小明
    console.log(Ming); //小明
    但如果像下列這樣：
    console.log(Ming); //undefined
    var Ming; //創造階段
    Ming = “小明”; //執行階段
    如果是函式陳述式，則不用擔心呼叫的位置，如下：
    callName(); // 呼叫小明
    function callName(){
    console.log(“呼叫小明”);
    }
    －－＞函式陳述式在創造階段，就已經包含記憶體完整內容
    相反地，如果是函式表達式，則要注意創造和執行階段：
    callName(); //callName is not a function.
    console.log(callName); //undefined
    var callName = function(){
    console.log(“呼叫小明”);
    }
    上述是因為，函式表達式會有兩階段產生（與變數相同的概念）
    //創造階段：
    var callName;
    //執行階段：
    callName = function(){
    console.log(“呼叫小明”);
    }
    callName();
    [比較一]：函式陳述式與函式表達式：
    function callName(){
    console.log(“呼叫小明 1”);
    }
    var callName = function(){
    console.log(“呼叫小明 2”);
    }
    callName(); //呼叫小明2
    如果將上述兩種函式調換位置：
    var callName = function(){
    console.log(“呼叫小明 2”);
    }
    function callName(){
    console.log(“呼叫小明 1”);
    }
    callName(); //呼叫小明2　　　結果相同
    　　//上述，結論為 呼叫小明2
    　　//創造階段：函式優先
    function callName(){
    　console.log(“呼叫小明 1”);
    }
    var callName;
    　　//執行階段
    callName = function(){
    console.log(“呼叫小明 2”);
    }
    callName();
    [比較二]：同樣函式陳述式：
    function callName(){
    console.log("小明");
    }
    callName(); //杰倫
    function callName(){
    console.log("杰倫");
    }
    callName(); //杰倫
    答案都是杰倫，原因在於兩個名稱相同的函式陳述式，後者會覆蓋前者
  2. Not defined 和undefined差別
    Undefined：記憶體已有空間給他，但未賦予值
    Not defined：沒有在記憶體上
    [注意]：勿將undefined賦予給變數（undefined是系統給予的值）
    　如果要賦予空值，請使用null

    (a) JavaScript是一種單執行續的程式語言
    (b) 單執行緒是一種針對系統的執行
    (c) 非同步(Async)是針對程式語言的本身－－＞此牽涉到事件佇列（Event Queue）
    非同步內容放到事件佇列，等到同步的程式碼執行完成，才執行佇列內容
